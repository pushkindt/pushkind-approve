# Supporting Pages and Integrations Specification

## Purpose

These features are not the primary order workflow, but they expose important reporting, communication, and integration behavior that affects operations.

## History

`/history/` is available to all non-`default`, non-`vendor` users. It displays `OrderEvent` records for the current hub filtered by a timestamp threshold. Supported relative periods come from `get_filter_timestamps()` and are relabeled for the UI. Events are ordered newest-first and rendered with the `EventType` enum for display text and colors.

## Help

`/help/` is available to all non-`default`, non-`vendor` users. It renders:

- an aggregate order summary table generated by raw SQL over orders, sites, and categories
- project-to-validator responsibility mapping
- category-to-validator responsibility mapping

Responsibility lists are based only on users with role `validator`.

## Support Requests

`POST /support/call/` sends an email to both `ADMIN_EMAIL` and the current user if the submitted comment length is between 1 and 2047 characters. Otherwise the request fails with a validation flash message.

## Email Notifications

`SendEmailNotification()` sends templated email to reviewers whose per-event preference flag is enabled. Supported event kinds in current use are `new`, `approved`, `disapproved`, and `comment`. Each email contains a JWT login link back to the target order page.

`LeaveCommentForm.comment_and_send_email()` can notify only a selected subset of reviewers and records a separate `notification` event when mail is sent.

## 1C Export Integration

`Prepare1CReport()` fills `app/static/upload/template1C.xlsx` with order, category, budget, site, initiative, quantity, vendor, and approval data. `/orders/excel1C/<id>` either downloads the generated workbook or emails it to `AppSettings.email_1C`. Approval can also auto-trigger this mail when the order becomes fully approved and `notify_1C` is enabled.

## Other Export Templates

- `/orders/excel1/<id>` uses `template.xlsx`
- `/orders/excel2/<id>` uses `template2.xlsx`
- `/orders/save/` and `/users/download` generate XLSX in memory without file templates

## Known Implementation Notes

- The help page uses interpolated SQL text with current hub ID.
- Support and notification email delivery depends on configured mail settings; failures are not wrapped in local retry logic.
- The app writes operational logs to `logs/app.log` only when Flask debug mode is off.
